# Get Help with Epicode

Welcome! This guide points you to the right place depending on what you need.
Epicode is an open-source project — community support happens on GitHub.

## Where to get help

| Need | Where to go |
|-----|-------------|
| **Ask a question** / get help using Epicode | [GitHub Discussions](https://github.com/sunormesky-max/epicode/discussions) |
| **Report a bug** or request a feature | [GitHub Issues](https://github.com/sunormesky-max/epicode/issues/new/choose) |
| **Report a security vulnerability** (private) | [Security Advisories](https://github.com/sunormesky-max/epicode/security/advisories/new) |
| **Read the docs** (architecture, API, deployment) | [epicode.cn/#/docs](https://epicode.cn/#/docs) |

## Before you ask

To get a fast, useful answer:

1. **Search first** — check existing [Discussions](https://github.com/sunormesky-max/epicode/discussions) and [Issues](https://github.com/sunormesky-max/epicode/issues) for your question.
2. **Pick the right channel** — questions go in Discussions, confirmed bugs go in Issues. If you're unsure, start in Discussions.
3. **Include context** when reporting a bug: the Epicode version (or commit), your OS, exact steps to reproduce, expected vs actual behavior, and any logs.

## Filing a good bug report

Use the issue template and fill in:

- Epicode version (`cat version.txt` or `epicode --version`)
- Environment (OS, Rust version, Node version, deployment method)
- Minimal reproduction steps
- Expected and actual behavior
- Relevant logs or screenshots

## Contributing

Want to contribute code, docs, or skills? Read the [Contributing Guide](CONTRIBUTING.md). We also have a [`good first issue`](https://github.com/sunormesky-max/epicode/labels/good%20first%20issue) label for newcomer-friendly tasks.

## Security reports

**Do not open a public issue for security vulnerabilities.** Report them privately via [Security Advisories](https://github.com/sunormesky-max/epicode/security/advisories/new) so they can be triaged before public disclosure. See [SECURITY.md](SECURITY.md) for the full policy.

## Response expectations

Epicode is maintained by a small team. We read every issue and discussion but may not respond immediately. PRs with tests and clear descriptions move fastest.
