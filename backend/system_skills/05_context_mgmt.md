# 上下文管理 (Context Management)

## 概述
你需要维护跨对话轮次的上下文连续性。本技能教你如何正确管理对话状态和用户上下文。

## 上下文层次

### 即时上下文（当前对话）
- 当前对话中的所有消息
- 用户刚才说了什么、你刚才回答了什么

### 会话上下文（最近历史）
- 使用 `memory_search` 搜索最近的交互记录
- 检查用户是否有活跃的偏好或习惯
- 了解用户当前关注的话题

### 长期上下文（持久记忆）
- 用户的身份信息、工作领域
- 用户的沟通风格偏好
- 历史重要决策和结论

## 上下文维护规则

### 每次对话开始
1. 获取 `stats` 了解当前记忆库状态
2. 如果用户提到特定话题，`memory_search` 查找相关历史

### 对话中
1. 新的重要信息 → 立即 `memory_create`
2. 引用之前的内容 → `memory_search` 确认准确
3. 用户纠正信息 → 存入新记忆 + 反馈旧记忆为irrelevant

### 对话结束前
1. 总结本次对话的关键信息
2. 存入对话摘要作为记忆
3. 对本次搜索结果批量提交反馈

## 最佳实践
- **宁可多存也不要漏存** — 无用记忆会被自动清理，但关键信息丢失无法恢复
- **标签一致性** — 每次用相同的标签指代相同的概念
- **时间标记** — 对时效性信息在content中包含日期
- **来源标注** — 在content中区分"用户说的"和"推理得出的"
