# 自动进化循环 (Auto-Evolution Loop)

## 概述
系统具备自我进化能力。你的每一次反馈都会让搜索变得更准确。本技能教你如何驱动这个进化循环。

## 反馈闭环

```
用户请求 → 搜索记忆 → 返回结果 → 你提交反馈 → 系统自动学习 → 下次搜索更准
```

## 如何提交反馈

使用 `feedback_submit` 工具，必须参数：
- **memory_ids**: 你要评价的记忆ID列表
- **relevance**: 这条记忆对你有多相关
  - `highly_relevant` — 完全相关，直接解决了问题
  - `partially_relevant` — 部分相关，有一点帮助
  - `irrelevant` — 不相关，没有帮助
- **outcome**: 使用这条记忆后的任务结果
  - `task_completed` — 任务顺利完成
  - `task_partial` — 部分完成
  - `task_failed` — 任务失败
- **notes** (可选): 自由文本备注
- **query** (可选): 原始搜索查询

## 反馈的效果
- 正面反馈 → 该记忆在未来的搜索中排名更靠前
- 负面反馈 → 该记忆排名下降
- 反馈数据本身也会被系统学习，优化整体搜索策略

## 执行规则

### 每次搜索后必须执行
1. 评估搜索结果是否解决了用户问题
2. 对每个有明确评价的结果提交 `feedback_submit`
3. 绝不要空反馈 — 这是系统进化的核心驱动力

### 对话结束时
- 对本次所有未反馈的搜索结果批量补交反馈
- 用 `session_summary` 总结本次会话
